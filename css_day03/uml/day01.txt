1. UML(Unified Modeling Language)
- UML은 시스템 분석, 설계, 구현 등 개발 과정에서 시스템 개발자와 고객의 상호 간의 의사소통을 원활하게 하기 위해 표준화한 대표적인 객체지향 모델링 언어이다.
- 크게 사물, 관계, 다이어그램으로 나뉘어진다.

2. UML의 사물
구조사물 - 시스템의 개념, 물리적인 요소를 표현
행동사물 - 시간과 공간에 따른 요소를 행위로 표현한 것
그룹사물 - 패키지 및 요소들을 그룹으로 묶어서 표현한 것
주해사물 - 부가적인 설명이나 제약조건을 표현한 것

3. UML의 관계
연관관계(Association)
	- 사물을 실선으로 연결하여 표현한다
	- 방향성은 화살표로 표현하며, 양방향 관계의 경우 실선으로 연결한다
	
집합관계(Aggregaion)
	- 하나의 사물이 다른 사물에 포함되어 있는 관계이다
	- 포함되는 쪽에서 포함하는 쪽으로 빈 마름모로 연결한다.

일반화관계(Generalization)
	- 하나의 사물이 다른 사물에 비해 더 일반적이거나 구체적인 관계이다.
	- 하위 사물에서 상위 사물 쪽으로 빈 화살표로 연결한다.
	
의존관계(Dependency)
	- 서로 연관은 있으나 필요에 의해 서로에게 영향을 주는 짧은 시간동안 연관을 유지하는 관계

4. 구조적 다이어그램의 종류
- 패키지 다이어그램(package) : 그룹화한 패키지의 관계 표현
- 클래스 다이어그램(class) : 클래스와 클래스 사이의 속성과 관계 표현
- 객체 다이어그램(object) : 객체와 객체사이의 관계 표현
- 컴포넌트 다이어그램(component) : 컴포넌트 간의 관계나 인터페이스 표현
- 배치 다이어그램(deployment) : 결과물, 프로세스, 컴포넌트 등 물리적 요소의 위치 표현
- 복합체 구조 다이어그램(composite structure) : 클래스나 컴포넌트의 복합 내부구조를 표현

5. 행위 다이어그램의 종류
- 유스케이스(use case) : 사용자의 요구를 분석하는 것으로 기능, 모델링 작업에 사용
- 시퀀스(sequence) : 상호 작용하는 시스템이나 객체들이 주고 받는 메세지를 표현
- 커뮤니케이션(communication) : 동작에 참여하는 객체들이 주고 받는 메세지와 객체들 간의 연관 관계를 표현
- 상태(state) : 하나의 객체가 자신이 속한 클래스의 상태 변화와 다른 객체의 상호작용에 따른 변화를 표현
- 활동(active) : 시스템이 어떤 기능을 수행하는지 로직과 조건에 따른 흐름을 순서로 표현
- 상호작용(interaction) : 상호작용 간의 제어 흐름 표현
- 타이밍(timing) : 객체 상태 변화와 시간 제약을 명시적 표현

유스케이스란?
- 요구사항을 사용자 중심 시나리오로 분석하여 흐름을 나타낸 것이다.
시스템의 동작을 모형화 시킨 것으로, 시스템의 외부에서 본 기능을 명확하게 정리해가며, 개발자와 사용자와의
상호작용을 표시한 것이다.

유스케이스의 기호
1) 시스템(System)
	- 개발하고자 하는 것 자체, 시스템 범위를 정의하여, 흐름이 일이 일어나는 영역의 경계를 의미한다.
2) 액터(Actor)
	- 시스템 외부에서 시스템과 상호작용하여 특정한 목적을 취하는 객체를 의미한다. 
	액터는 반드시 하나 이상의 유스케이스들과 상호작용 해야한다.
	ex) 소비자, 사용자, 관리자 ...
3) 유스케이스(Use Cases)
	- 시스템 내에서 일련의 작업을 수행하기 위한 행위들을 의미한다.

관계의 표현
1) 연관 관계(Association) 
	- 유스케이스와 액터 사이에 상호작용이 있다는 뜻으로, 실선으로 표시한다.
2) 포함 관계(Include)
	- 하나의 유스케이스가 실행될 때 포함 관계에 있는 유스케이스가 반드시 실행되어야 한다.
	ex) 로그인 시 비밀번호 확인
3) 확장 관계(Extend)
	- 확장 관계는 두 개의 유스케이스 간의 확장성을 표시한다
	- 하나의 유스케이스가 실행될 때 포함 관계에 있는 유스케이스가 특정 상황에서만 실행된다.
4) 일반화 관계(Generalization)
	- 일반화 관계는 부모 유스케이스와 자식 유스케이스들 간의 상속관계를 표시한다.
	- 일반화된 객체로 비어있는 삼각형으로 표시한다.
